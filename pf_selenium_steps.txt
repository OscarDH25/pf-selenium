PF_SELENIUM – LOG DE PASOS DEL PROYECTO
=====================================

OBJETIVO DEL PROYECTO
--------------------
Crear un proyecto de portfolio QA no-junior con:
- App fullstack
- Automatización E2E con Selenium (Java)
- Docker + CI
- Enfoque en testabilidad, estabilidad y legibilidad


FASE 0 – PREPARACIÓN DEL ENTORNO (WINDOWS)
------------------------------------------

Software instalado:
- Git
- IntelliJ IDEA
- JDK 17 (Eclipse Temurin)
- Docker Desktop
- Node.js + npm
- Google Chrome

Configuraciones clave:
- JAVA_HOME y PATH configurados por el instalador
- PowerShell ExecutionPolicy cambiado a RemoteSigned (CurrentUser)
- Git configurado con user.name y user.email
- Rama por defecto de Git fijada a main
- Virtualización activada en BIOS (SVM Mode en ASUS B550)

FASE 2 – DOCKER COMPOSE BÁSICO (SKELETON)
-----------------------------------------

Objetivo:
Verificar que el proyecto es portable y ejecutable con Docker.

docker-compose.yml inicial:
- Servicio web con nginx:alpine
- Puerto local 8080 → container 80

Comandos clave:
- docker compose up -d
- docker compose ps
- docker compose down

Resultado:
- nginx accesible en http://localhost:8080
- Docker funcionando correctamente

PR:
"chore: add docker compose skeleton"
Mergeado a main


FASE 3 – DOCKER + POSTGRES
--------------------------

Objetivo:
Añadir base de datos real con persistencia.

Cambios en docker-compose.yml:
- Servicio db con postgres:16-alpine
- Variables de entorno:
  - POSTGRES_DB=pf
  - POSTGRES_USER=pf_user
  - POSTGRES_PASSWORD=pf_pass
- Volumen persistente: pf_db_data
- Puerto 5432 expuesto

Verificación:
- docker compose exec db psql -U pf_user -d pf -c "SELECT version();"
- Comprobada persistencia tras down/up

Resultado:
- Postgres funcionando correctamente en Docker
- Infraestructura lista para backend real

PR:
"chore: add postgres service to docker compose"
Mergeado a main
